1システム概要

Twicome は Twitch の VOD コメントをユーザ単位で検索・分析する Web アプリです。 取得済みコメントに対して、通常のキーワード検索だけでなく意味ベクトル検索・感情検索・統計分析・クイズまで幅広く活用できます。

コメント閲覧 VOD・配信者・キーワード・除外ワードで絞り込みながら快適に閲覧できます。
意味検索 FAISS ベクトル検索で、表現が違っても意味が近いコメントを探索できます。
感情検索 笑い・驚き・称賛など6軸のスライダーを合成して、狙ったトーンのコメントを抽出します。
コミュニティノート コメントに AI 判定ステータス・危険度スコア付きの注釈が表示されます。
統計分析 時間帯・曜日分布・配信者別活動率・影響度などを多角的に可視化します。
クイズ 指定ユーザの発言かどうかを当てる二択ゲームです。スコアと正答率が記録されます。

3トップページ

基本操作

  1. ドロップダウンで Twitch ユーザを選択（名前で検索可）
  2. 「コメント一覧を表示」でコメントページへ遷移
  3. 「統計ページ」「クイズで遊ぶ」も選択ユーザに連動して遷移先が切り替わる

補助導線

  • 固定ショートカットから直接コメント一覧へ移動できます
  • 「配信者を追加」「使い方ガイド」へのリンクがあります

人気コメントランキング

いいね + わるいね の合計スコア上位 20 件を表示します。 各行をクリックすると対象コメントの cursor 表示に移動します。

4コメント一覧ページ

/u/{ユーザ名}?platform=twitch

コメントカードで見られる情報

  • VOD: Twitch の該当時刻へジャンプ
  • YouTube: 対応 URL がある場合のみ表示。該当秒にジャンプ
  • VODタイトル / 配信者 / 投稿者: どの文脈の発言か把握できます
  • オフセット: 配信内の位置（hh:mm:ss）
  • 投稿時刻: JST 表示 + 相対時間（24時間以内は赤表示）
  • bits: Bits 付きコメントをバッジ表示
  • 本文: Twitch エモートは画像として描画されます

ページング挙動

  • 下までスクロールすると次ページを自動読み込み（無限スクロール）
  • 「前の投稿も読む」で前ページを先頭に追加読み込み

5フィルター/並び替え

項目 内容
VOD 特定 VOD のコメントのみに限定
配信者 特定チャンネルのコメントのみに限定
本文検索 (q) 部分一致のキーワード検索（大文字・小文字区別なし）
除外ワード (exclude_q) 指定語を含むコメントを除外。空白・カンマ・読点「、」区切りで複数指定可
1ページ表示件数 20 / 50 / 100 / 200 から選択

値 意味

randomランダム順

6特殊検索（類似/典型度/感情）

類似検索

  • 入力文と意味が近いコメントを検索します（表現が違っても意味が近いものを発見できます）
  • 件数は 20 / 50 / 100 から選択
  • 結果に「類似度」スコアが表示されます

典型度スライダー（重心検索）

  • 0% に近いほど「そのユーザらしい・典型的な発言」
  • 100% に近いほど「普段と違う・珍しい発言」
  • 結果には「重心類似度」が表示されます

感情スライダー

  • 笑い・驚き・称賛・怒り・悲しみ・応援を 0〜100 で合成
  • 1軸も有効にしていない状態では検索できません
  • 一致度バッジ付きで結果表示。リセットボタンで全スライダーをリセット
類似・典型度・感情のいずれかを実行中は特殊モードになります。 この間は通常の無限スクロールは停止し、「通常表示に戻す」ボタンで通常一覧へ戻ります。
類似検索・典型度検索・感情検索にはユーザごとの FAISS インデックスが必要です。 インデックス未作成のユーザでは利用できません。

7リアクション

いいね わるいね

ボタンを押すと画面上のカウントが即時増加し、裏側では同一コメント・同一種別の連打を 500ms 単位でまとめて送信します（count パラメータ）。

このリアクション値はランキングや並び替え、コミュニティノート運用に使われます。

8コメント個別ページ（cursor）

/u/{ユーザ名}?cursor={コメントID}
  • コメントカード全体をクリックするとこの表示に遷移します
  • 対象コメントが黄色でハイライト表示されます
  • 同一 VOD のコメントをまとめて表示し、前後の文脈を確認できます
  • 対象ユーザの発言はグレー背景で区別されます
  • URL を共有することで、特定の発言を直接参照できます
cursor 指定時は、通常のフィルター条件より「対象コメントがある VOD の表示」が優先されます。

9コミュニティノート

条件を満たしたコメントに、注釈テキスト・判定ステータス・スコアが表示されます。 AI による自動審査で付与されます。

判定ステータス

内部値表示名意味
supported 裏付けあり 内容に根拠・裏付けがある
insufficient 情報不足 判断するには情報が不十分
inconsistent 矛盾あり 事実と矛盾する情報が含まれる
not_applicable 該当なし コミュニティノート対象外

危険度スコア

危険度は次の4軸平均で算出されます。

(被害可能性 + 誇張度 + 根拠不足 + 主観度) ÷ 4
0–29: 低 30–59: 中 60–100: 高

検証可能性を含む5軸スコアバーもコメント単位で表示されます。

10統計ページ

/u/{ユーザ名}/stats?platform=twitch

表示内容

  • 書き込み時間分布（JST 24時間・棒グラフ）
  • 曜日分布（ドーナツチャート）
  • 配信者ごとのアクティブ状況（上位 50）
  • コメント影響度分析（条件を満たす場合のみ）
  • コミュニティノート分析（ノートがある場合のみ）

コミュニティノート分析

  • 危険度ヒストグラム（10点刻み）
  • 5軸平均のレーダーチャート
  • 判定ステータス分布（裏付けあり / 情報不足 / 矛盾あり / 該当なし）

配信者ごとのアクティブ状況

配信時間を5分区間に分け、対象ユーザが書き込んだ区間を活動・書き込んでいない区間を非活動として比率表示します。 テーブルの見出し行をクリックするとその列でソートできます。

コメント影響度分析

活動区間と非活動区間で、他コメンターのコメント数・人数を比較します。 有意性は Mann-Whitney U 検定で表示されます（*** / ** / * / n.s.）。

影響度分析は対象ユーザの関与 VOD 数が 500 以下の場合に計算されます。データ不足時は表示されません。

11コメント当てクイズ

/u/{ユーザ名}/quiz?platform=twitch
  • 30問固定・ライフ3、3ミスで終了
  • 「対象ユーザの発言」か「他の人の発言」かを二択で回答
  • 正解で +1 点、不正解時は投稿者名を表示
  • 終了時にスコア・正答率・最大連続正解を表示
  • 問題は対象ユーザ発言と他ユーザ発言を半々で構成（同じ配信集合から抽出）
問題セットは開始時に API で生成されます（対象ユーザ発言と他ユーザ発言を混在）。

12配信者追加

/add_user
  1. Twitch ユーザ名またはチャンネル URL を入力
  2. 追加を実行すると、存在確認後に監視対象 CSV へ登録
  3. 同ページ下部に現在の監視対象一覧を表示
追加直後にコメントが増えるわけではありません。定期バッチの取得タイミングで反映されます。

14API リファレンス

メソッド パス 用途
GET /api/u/{login} コメント一覧取得（vod_id / owner_user_id / q / exclude_q / sort / cursor など対応）
GET /api/u/{login}/similar 類似検索（q, top_k）
GET /api/u/{login}/centroid 典型度検索（position=0.0-1.0, top_k）
GET /api/u/{login}/emotion 感情検索（joy / surprise / admiration / anger / sadness / cheer）
GET /api/emotion_axes 感情軸定義の取得
POST /like/{comment_id}?count=1 いいね加算（1〜100）
POST /dislike/{comment_id}?count=1 わるいね加算（1〜100）
GET /api/u/{login}/quiz/start クイズ問題生成（count=10-100）
ブラウザ外から POST /like / POST /dislike を呼ぶ場合は、 X-Requested-With: XMLHttpRequest ヘッダーを付けてください。
